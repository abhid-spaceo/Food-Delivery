# Phase 1.5 — Design System Foundation + Visual Retrofit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax. **Do NOT commit — leave changes for the human to review and approve each commit** (user rule).

**Goal:** Make the app *look* production-grade by introducing a shared design system that matches the hi-fi mockups (`docs/food-delivery/design/*.html`) and retrofitting the existing auth/restaurant/admin screens to it — visual only, no behavior change.

**Architecture:** The app uses shadcn/ui on Tailwind v4 with CSS-variable theme tokens in `app/globals.css` consumed via `@theme inline`. The highest-leverage change is the **token layer**: swapping the default neutral palette for the mockups' brand palette (coral `#FF3B5C`, off-white surfaces, soft borders) propagates through every shadcn component automatically. Then we polish the shared shells (header, sidebars), KPI cards, and tables, and do a per-screen sweep. From Phase 2 onward, new screens are built on this system.

**Tech Stack:** Tailwind v4 · shadcn/ui · Next.js 16 (`next/font/google`) · lucide-react. No new dependencies.

**Design source of truth:** `docs/food-delivery/design/index.html` (palette), `admin-design.html` (admin console), `restaurant-design.html` (partner portal), `customer-design.html` (customer web). Palette tokens from `index.html`: `--ink:#15141A · --text:#46454F · --muted:#8E8D99 · --line:#ECEBF0 · --bg:#F6F5F8 · --card:#FFFFFF · --brand:#FF3B5C · --brand-dark:#E82E4D · --brand-soft:#FFEDF0`, font **Inter**, radius ~18px, soft shadows.

**Branch:** `Abhi/qwikbite`. **Critical constraint:** this is a *visual* retrofit — **do not change behavior, routes, data, or the text/accessible names that E2E tests query** (`"Sign in"`, `"Accept"`, `"Start preparing"`, `"Mark ready"`, status labels like `"Placed"`/`"Ready"`, the `"Open"` links, `"No further actions for this order."`). After each task, the existing test suite must stay green.

**Preconditions:** Phase 1 complete (branch `Abhi/qwikbite`); `pnpm test && pnpm build && pnpm test:e2e` currently green; dev DB seeded.

**Verification model:** Visual tasks are verified by (a) `pnpm build` + `pnpm lint` clean, (b) the full test suite still green (no regression), and (c) a **browser comparison** against the corresponding mockup. There are no new unit tests (this is styling).
